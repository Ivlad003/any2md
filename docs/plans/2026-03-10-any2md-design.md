# any2md — Design Document

## Overview

CLI utility in Rust for converting files to Markdown. Extensible trait-based plugin architecture where each input format implements a `Converter` trait. First converter: PDF.

**Target audience:** Technical documentation (books, docs with code blocks, headings, lists, diagrams).

## Architecture

```
AnyToMarkdown CLI
├── Core
│   ├── trait Converter { fn convert(&self, input, options) -> Document }
│   ├── Document (unified intermediate model)
│   └── MarkdownRenderer (Document → .md)
├── Converters
│   ├── PdfConverter (impl Converter)
│   ├── HtmlConverter (future)
│   └── AudioConverter (future)
└── CLI (clap, auto-detects format by extension)
```

All converters produce a unified `Document` model. One `MarkdownRenderer` generates Markdown from it. New format = new `impl Converter`, nothing else changes.

## Intermediate Model: Document

```rust
pub struct Document {
    pub metadata: Metadata,
    pub pages: Vec<Page>,
}

pub struct Metadata {
    pub title: Option<String>,
    pub author: Option<String>,
    pub date: Option<String>,
}

pub struct Page {
    pub elements: Vec<Element>,
}

pub enum Element {
    Heading { level: u8, text: String },
    Paragraph { text: RichText },
    CodeBlock { language: Option<String>, code: String },
    List { ordered: bool, items: Vec<ListItem> },
    Table { headers: Vec<String>, rows: Vec<Vec<String>> },
    Image { data: Vec<u8>, alt: Option<String> },
    HorizontalRule,
    BlockQuote { text: RichText },
}

pub struct RichText {
    pub segments: Vec<TextSegment>,
}

pub struct TextSegment {
    pub text: String,
    pub bold: bool,
    pub italic: bool,
    pub code: bool,
    pub link: Option<String>,
}

pub struct ListItem {
    pub text: RichText,
    pub children: Vec<ListItem>,
}
```

## Converter Trait & CLI Options

```rust
pub trait Converter {
    fn name(&self) -> &str;
    fn supported_extensions(&self) -> &[&str];
    fn convert(&self, input: &Path, options: &ConvertOptions) -> Result<Document>;
}

pub struct ConvertOptions {
    pub image_mode: ImageMode,
    pub page_mode: PageMode,
    pub image_output_dir: PathBuf,
}

pub enum ImageMode {
    Extract,  // save to folder, reference in markdown
    Inline,   // base64 embedded
}

pub enum PageMode {
    SingleFile,
    SplitPages,
}
```

**CLI:**

```
any2md <input> [options]

Options:
  -o, --output <path>        Output file/directory (default: input_name.md)
  --images <extract|inline>  Image mode (default: extract)
  --pages <single|split>     Page mode (default: single)
  -h, --help                 Help (auto-generated by clap)
```

## PDF Converter — Heuristic Pipeline

```
PDF file
  │
  ▼
┌─────────────────┐
│  1. Extraction   │  lopdf: extract raw elements
│  (RawElement)    │  - text blocks with coords, font, size
│                  │  - images (Image XObjects)
└──────┬──────────┘
       ▼
┌─────────────────┐
│  2. Analysis     │  Heuristics classify each block
│  (Classifier)    │
└──────┬──────────┘
       ▼
┌─────────────────┐
│  3. Assembly     │  Classified elements → Document
│  (Assembler)     │
└─────────────────┘
```

### Classification Heuristics

| Signal | Rule |
|--------|------|
| **Heading** | Font size > document average + bold |
| **Code** | Monospace font (Courier, Consolas, etc.) |
| **List** | Line starts with `•`, `-`, `1.`, `a)` or has characteristic indent |
| **Table** | Group of blocks with aligned X-coordinates in multiple columns |
| **Paragraph** | Everything else (fallback) |
| **Bold/Italic** | Font name contains "Bold"/"Italic" |

Priority order: code → heading → list → table → paragraph.

## Markdown Renderer

| Element | Markdown output |
|---------|----------------|
| `Heading { level: 2 }` | `## Text` |
| `Paragraph` | Text with inline formatting + blank line |
| `CodeBlock { lang: "rust" }` | ` ```rust\ncode\n``` ` |
| `List { ordered: false }` | `- item\n- item` |
| `Table` | `\| h1 \| h2 \|\n\|----\|----\|\n\| c1 \| c2 \|` |
| `Image` (extract) | Save PNG to folder → `![alt](images/img_1.png)` |
| `Image` (inline) | `![alt](data:image/png;base64,...)` |
| `HorizontalRule` | `---` |
| `BlockQuote` | `> text` |

**RichText:** bold → `**text**`, italic → `*text*`, code → `` `text` ``, link → `[text](url)`

**Page separation (SingleFile):** `---` with `<!-- page N -->` comments.

## Error Handling

```rust
pub enum ConvertError {
    FileNotFound(PathBuf),
    UnsupportedFormat(String),
    CorruptedFile(String),
    ImageExtractionFailed(String),
    IoError(std::io::Error),
}
```

### Edge Cases

| Situation | Behavior |
|-----------|----------|
| Empty page | Skip |
| Unknown image format | Log warning, skip |
| Password-protected PDF | Return error with message |
| Unrecognized font | Classify as paragraph (fallback) |
| Large PDF (1000+ pages) | Process page by page, don't load all into memory |
| Nested lists | Detect by indentation, up to 3 levels |
| Table without clear borders | Best effort; if heuristic unsure, render as text |

### CLI Output

- Progress to stderr: `Converting page 3/15...`
- Warnings to stderr: `Warning: skipped unsupported image on page 5`
- Result to stdout or file
